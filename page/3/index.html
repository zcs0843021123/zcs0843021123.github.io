3. 在自己的github上创建一个仓库

仓库的名字就叫做 your_github_user_name.github.io
your_github_user_name.github.io 这个名字的格式是固定的 不可以谁便乱改
这个仓库一定要是空的，什么都没有，否则deploy不上去（deploy调用的就是 git push）

有时hexo deploy不上去，不过有时候也不报错，具体表现为博客不更新或者是报404，是因为
1）你的GitHub的仓库里有东西，push不上去。
2）你没有给git设置upstream

- 执行hexo generate之后，在 .deploy_git 里边就是你要发布的博客。
  先用 git remote -v 查看你的这个git仓库有木有远程仓库，一般来说没有
  远程仓库 如: https://github.com/your_github_user_name/your_github_user_name.github.io
- 关联本地库
  git remote add origin https://github.com/your_github_user_name/your_github_user_name.github.io
  添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。
- 推送当前分支并把远程设置成 upstream
  git push --set-upstream origin master
